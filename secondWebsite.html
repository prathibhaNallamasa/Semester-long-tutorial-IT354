Functions

Named Functions

Named functions are nothing but the functions which have names. Named functions can be used while declaring the functions or even used in the function expressions. 

               
                   function product(a,b)
                   {
                      c = a * b ;
                       console.log("the product of the two numbers is" , c)
                   }
                   console.log(product(6,7))

                   // returns --- the product of the two numbers is 42

Function Expressions

A function can be declared using a function expression. In this method, the function is assigned to a variable like var a = function(a,b){ return a+b; } The function can be called using the assigned variable.

                
                    var x = function RectangleArea(a,b) {
                        y = a * b;
                        console.log("The area is", y );
                      }
                      console.log(x(5,8));

                      // returns --- The area is 40

Arrow Functions

Arrow functions are just the shorter syntax for declaring a function. This method includes a token " =>" which is known as the fat arrow to declare the functions. There is no need for a function name, if using this method.

                
                    var x = (a) => {
                        y = a * a;
                        console.log("The area is", y );
                      }
                      console.log(x(6));

                      // returns --- The are is 36

Other topics

Optional arguments

JavaScript ignores all the extra arguments which we pass through the function. And if do not pass enough, then the missing values are undefined. These undefined values are filled with default values. 

                
                    function multiply(a, b=1) {
                        return a * b
                      }
                      multiply(3)
                      // returns -- 3

Rest Parameters

Rest parameter is an improved way to handle function parameter, allowing us to more easily handle various input as parameters in a function. The rest parameter syntax allows us to represent an indefinite number of arguments as an array. When we use "...something" in a parameter, it gets bound to an array. this means that we can get a list and keep on iterating it. 

                
                    function add(...numbers) {
                        let result = 0;
                        for(let number of numbers)
                        {
                          result += number;
                        }
                        return result;
                      }
                      add(100,10,20,30);


                      // returns 160

Nested scope

Functions have their own scope and hence, the outer functions cannot access the variables in an inner function. where as, the inner functions can access the variables in the outer function.

                
                    function add(a,b)
                    {
                    function addition()
                    {
                    total = a+b;
                    return total;
                      }
                    return addition()
                    }
                    add(80,20)

                    // returns 100
