API

what is an API

API is acronym for Application Programming Interface. API is a list of operations which can be used by developers and also have a description of what they do. API is a set of routines, protocols, and tools for building software applications. Basically, an API specifies how software components should interact. Additionally, APIs are used when programming graphical user interface (GUI) components.


Rest API

REST is acronym for REpresentational State Transfer. It is architectural style for distributed hypermedia systems and was first presented by Roy Fielding in 2000.
Six guiding constraints define a RESTful system. The formal REST constraints are as follows:

Client-server architecture:

The uniform interface separates clients from servers. Servers are not concerned with the user interface or user state, so that servers can be simpler and more scalable.

Stateless:

Each request from client to server must contain all of the information necessary to understand the request, and cannot take advantage of any stored context on the server. session is totally kept on client.

Cacheable:

clients can cache responses. Responses must therefore, implicitly or explicitly, define themselves as cacheable, or not, to prevent clients reusing stale or inappropriate data in response to further requests.

Layered system:

The architecture is layered hierarchically so that the components cannot see beyond their immediate layer with which they are interacting. Intermediary servers may improve system scalability by enabling load-balancing and by providing shared caches.

Uniform interface:

The uniform interface constraint defines the interface between clients and servers. REST is defined by four interface constraints: identification of resources; manipulation of resources through representations; self-descriptive messages; and, hypermedia as the engine of application state.

Code on demand:

REST allows client functionality to be extended by downloading and executing code in the form of applets or scripts.

Postman

Postman is an application to test the API. It is an interface which allows HTML requests without writing any code. It comes with different options like GET, POST, ect. we just need to plug the route in the address bar and select the response method from the dropdown list. we can also give the API key in the headers section. There is also an option to select the response fromat for the request.

Example:

Below is the example of how an API is called in javascript and how the API returns the data.

                      
                        var XMLHttpRequest = require("xmlhttprequest").XMLHttpRequest;
                        let request = new XMLHttpRequest();
                        
                        request.open('GET', 'https://ghibliapi.herokuapp.com/films')
                        
                        request.responseType = 'json';
                        request.send()
                        request.onload = function() {
                          
                          var data = JSON.parse(this.responseText)
                            data.forEach(movie => {
                              console.log(movie.title)
                            })
                        }
