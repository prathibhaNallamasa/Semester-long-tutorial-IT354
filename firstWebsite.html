Tutorial for topics covered in week 2

Strings

Strings can be declared using 'single quotes' ('') , "double quotes" ("").
we can use quotes inside a string, as long as they don't match the quotes surrounding the string.

-> " My name is Prathibha" // returns --- My name is Prathibha
-> 'I am a grad student' // returns --- I am a grad student
-> "I am doing masters in 'Information Systems'" // --- I am doing masters in 'Information Systems'


Backticks helps in the output.(``) backticks helps to print a variable within a string.

var a = "Prathibha"
`My name is ${a}` // return ---- My name is Prathibha

Unary Operators

TYPEOF()

typeof() operator returns the type of the given argument. It can be a number or a string.

-> typeof(3.9) // returns --- number
-> typeof(6) // returns --- number
-> typeof("pink") // returns --- string

Unary plus

It can convert all string representations of numbers, boolean values(true and false) and null to numbers.
If the operand cannot be converted into a number, the unary plus operator will return NaN.

-> +3 // returns --- 3
-> + -9 // returns --- -9
-> +'4' //returns --- 4
-> + true // returns --- 1
-> + false // returns --- 0
-> + null // returns --- 0 ( as null is equal to zero, it returns zero)

Unary minus

It works similar to a unary plus operator but returns a negative value.

-> -3 // returns ---> -3
-> --9 // returns ---> 9
-> -'4' //returns ---> -4
-> - true // returns ---> -1
-> - false // returns ---> -0
-> -null // returns ---> -0

Logical not (!)

Returns false if its single operand can be converted to true ; otherwise, returns true .

-> var m = false
-> !m // returns true
-> var n = true
-> !n // returns false
-> !'green' // returns false (since there is a string inside the quotes,and string is considered as a truthy value)
-> !'' // returns true (an empty string will have falsy value)

Double Logical Not (!!)

It always gives the opposite of a logical not.
let us consider the above examples again to see the difference between both the operators.

-> var m = false
-> !!m // returns false
-> var n = true
-> !!n // returns true
-> !!'green' // returns true
-> !!'' // returns false

Increment operator(++)

The increment operator can be used before or after the operand.
If the operator is used after the operand, the value is returned before the operand is used.
If the operator is used before the operand, the value is incremented before returning.

-> a = 2
using after the operand
a++ ; // returns 2
a; // returns 3 as a is incremented in above step
-> a = 5
Using before the operand
++ a ; // returns 6 (a is incremented in this step)

Decrement operator

This works similar to increment operator and this can also be used before or after the operands.
This operator decrements the variable by 1;

-> a = 2
using after the operand
a-- ; // returns 2
a; // returns 1 as a is decremented in above step
-> a = 5
Using before the operand
-- a ; // returns 4 (a is decremented in this step)

Delete

This returns true if the value is deleted or even if it does not exist
But the operator does not work on variables and functions.

a = 5;
delete a // returns true
-> var c = 10;
c; // returns 10
delete c ; // returns false ( as c is a variable, it does not get deleted.)


Hoisting

Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their scope before code execution.
Inevitably, this means that no matter where functions and variables are declared, they are moved to the top of their scope regardless of whether their scope is global or local.
It is when a browser/ engine compiles variables and named functions immediately.
